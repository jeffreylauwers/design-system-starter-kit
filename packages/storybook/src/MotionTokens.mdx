# Motion

Motion tokens beheren timing en easing op één centrale plek. Alle componenten in het design system gebruiken deze tokens — waardoor `prefers-reduced-motion` automatisch en consistent wordt gerespecteerd.

## Transition Duration

Duur bepaalt hoe lang een transitie duurt. De preview laat de snelheid visueel zien — langzamere tokens hebben een langere sweep.

Bij `prefers-reduced-motion: reduce` worden alle duration-tokens naar `0ms` gezet via een centrale media query in het theme CSS-bestand. Componenten hoeven zelf geen media query te implementeren.

### Gebruik

| Token | Use case |
|---|---|
| `instant` | State-changes die onmiddellijk moeten aanvoelen |
| `fast` | Subtiele hover-states, kleurveranderingen |
| `normal` | Standaard UI-transitions — huidige standaard voor alle componenten |
| `slow` | Grotere bewegingen, accordions, details |
| `slower` | Complexe animaties, page-level transitions |

## Transition Easing

Easing bepaalt het acceleratieprofiel van een transitie. De preview toont een bolletje dat beweegt met die specifieke curve — zo is het verschil tussen curves direct zichtbaar.

### Gebruik

| Token | Use case |
|---|---|
| `default` | Algemeen — vergelijkbaar met CSS `ease` |
| `enter` | Elementen die het scherm binnenkomen (decelereren aan het einde) |
| `exit` | Elementen die het scherm verlaten (accelereren) |
| `move` | Elementen die van positie wisselen |
| `linear` | Progress bars, loaders, continue animaties |

## prefers-reduced-motion

Het design system implementeert `prefers-reduced-motion` via een centrale media query in het gegenereerde theme CSS-bestand:

```css
@media (prefers-reduced-motion: reduce) {
  :root {
    --dsn-transition-duration-instant: 0ms;
    --dsn-transition-duration-fast: 0ms;
    --dsn-transition-duration-normal: 0ms;
    --dsn-transition-duration-slow: 0ms;
    --dsn-transition-duration-slower: 0ms;
  }
}
```

Omdat alle componenten `var(--dsn-transition-duration-*)` gebruiken, worden animaties automatisch uitgeschakeld zodra een gebruiker reduced motion heeft ingesteld. Componenten hoeven zelf geen `@media (prefers-reduced-motion)` te implementeren.

Easing-tokens worden niet aangepast bij reduced motion — ze zijn niet relevant als er geen beweging is.
